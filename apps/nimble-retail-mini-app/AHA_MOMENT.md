# THE AHA MOMENT REQUIREMENT

This application's success is NOT measured by:

* Number of charts
* Number of features
* Number of filters
* Number of retailers analyzed

Success is measured by whether a prospect experiences an immediate insight that changes how they think about their category.

The application should consistently generate moments like:

"Wow, I didn't realize Premier Protein owned that much visibility."

"I didn't know sponsored products made up nearly half the shelf."

"I didn't realize Amazon and Walmart looked so different."

"I wonder how my brand compares."

That curiosity should naturally lead to:

"I want to run my category."

And ultimately:

"I want to talk to Nimble."

Every screen should be designed to maximize discovery and curiosity.

---

# INTERACTIVITY PRINCIPLES

Users should never feel like they are looking at a static dashboard.

The application should feel alive.

Examples:

* Click a category → instantly see insights
* Click a brand → instantly see competitive analysis
* Click a retailer → instantly see how the shelf changes
* Hover a brand → reveal additional intelligence
* Ask a question → receive an answer immediately

Every interaction should reveal something interesting.

---

# INSIGHTS BEFORE ANALYTICS

Never start with charts.

Never start with tables.

Never start with filters.

Start with:

1. What happened?
2. Why does it matter?
3. What should I do?

Only then show the supporting analytics.

---

# DISCOVERY-FIRST EXPERIENCE

The user should continuously uncover new insights.

Examples:

"Premier Protein owns Walmart."

↓

User clicks Premier Protein.

↓

"Premier Protein ranks organically, not just through paid placements."

↓

User clicks Organic.

↓

"Quest has higher ratings but significantly less visibility."

↓

User clicks Quest.

↓

"Quest has an opportunity to gain market share."

Every click should reveal another layer of intelligence.

---

# CONFERENCE TEST

If a conference attendee spends 30 seconds in the experience, they should uncover at least three surprising insights without needing instructions.

If they cannot, the UX should be redesigned.

---

# THE GOLDEN QUESTION

Every feature should answer:

"Does this increase the chance that a prospect says wow?"

If not, remove it.
